# UI/UX Design Principles — LeanSpirited Standard

These apply across all LeanSpirited products. Adapt the
"professional user test" to the specific target persona per project.

---

## Core principle

The user's target audience determines the quality bar.
If our user presents our tool's output to their own clients,
our UI must meet the standard their clients expect.

---

## Precision over decoration

Functional, refined, no unnecessary ornament.
Every element earns its place.

---

## Colour as signal, not wallpaper

Dominant neutral backgrounds. Brand colour as accent only:
active states, primary CTAs, key callouts.
Never use brand colour as a background flood.

---

## Typographic intention

Choose a type pairing and commit to it per product.
Display/heading font for hierarchy. UI font for body and controls.
Never introduce ad-hoc fonts mid-product.
Document the pairing in the project's ADR or README.

---

## Instinctive hierarchy

Primary action always dominant and visible without scrolling.
Navigation always findable. Secondary actions discoverable.
Never make the user think about where to look.

---

## Details that professionals notice

Pixel-perfect alignment. Consistent spacing units.
Hover states that respond crisply. Loading states that don't jar.
Error states that are calm and helpful.
These are not polish — they are the signal that the tool is trustworthy.

---

## Responsive by default

Design for the smallest likely screen first.
Test mentally at 375px, 768px, 1280px before shipping.
Touch targets 44px minimum.

---

## The professional user test [adapt per project]

Before shipping any UI: "Would our target user trust this to
present to their own clients?"
If the answer is uncertain, it's not ready.
